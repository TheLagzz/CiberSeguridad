 La identificación y clasificación son cosas distintas. La identificación es para identificar y enumerar las vulnerabilidades por lo general las más conocidas y cada sistema le asigna una identificación única que registra en su base de datos, las mas comunes son las siguientes:
 - **CVE (Common Vulnerabilities and Exposures):** Es como un diccionario que proporciona definiciones para las vulnerabilidades y exposiciones de seguridad pública, cada vulnerabilidad cuenta con un ID único que el formato de esta es "CVE-AÑO-NÚMEROSECUENCIAL"
	 - Ejemplo: CVE-2014-0160 el nombre popular de este fallo es "**Heartbleed**" que fue una vulnerabilidad critica descubierta en 2014 que afectaba la biblioteca de cifrado OpenSSL que es utilizada mundialmente para proteger conexiones web. Por lo que si me encontrara un sistema que utiliza una versión vulnerable de OpenSSL podría ser totalmente innecesario buscar por donde atacarlo, simplemente podría buscar un exploit que sea compatible con CVE-2014-0160 y se ahorra mucho tiempo al documentarlo.
- CWE (Common Weakness Enumeration): Es un estandar para el diseño del software y es mas general que CVE, lo que hace CWE lo que hace es proporsionar una categoria general que engloba vulnerabilidades y se les identifica con la etiqueta de **CWE-ID**.
	- Ejemplo: **CWE-89** que pertenece a la categoria de inyecciones SQL y lo que haría CVE identificaría cada vulnerabilidad, por lo que CVE -> Especifico & CWE -> General. Se puede destacar que CVE sirve para buscar exploits y CWE es para tener una visión mas amplia y el trasfondo que tiene una vulnerabilidad en su categoria y podría servir para los informes.
- CAPEC(Common Attack Pattern Enumeration and Classification): Es una lista para tener una visión de que patrones de ataque comunes que siguen los ciberdelincuentes pueden usar para explotar vulnerabilidades. En vez de enlistar o identificar las vulnerabilidades lo que hace es mostrar las estrategias o modos de operación que tienen los ciberdelincuentes, pero CAPEC si le asigna un ID único a las categorías de ataque y el formato es CAPEC-ID.
	- Ejemplo: CAPEC-66 es un patrón de ataque que aprovecha vulnerabilidades que permiten inyectar código a través de ella y se puede adaptar a escenarios en especifico, como el CWE-89 que también es de inyección SQL. 

Existen otro sistema de identificación de vulnerabilidades que es proveniente de Microsoft, son boletines de seguridad que nos dan en productos de la misma empresa. Su formato es MS(Año)-(ID).
- Ejemplo: El boletín MS17-010 fue abordó una serie de vulnerabilidades criticas en el protocolo SMBv1 (puerto TCP 445). Tenia vulnerabilidades y se podía explotar con un exploit llamado EternalBlue, algunas de las vulnerabilidades que estaban incluidas en este boletín se les asigno su correspondiente CVE por lo que se podrían tener varias vulnerabilidades identificadas por diferentes proveedores. Microsoft engloba varias vulnerabilidades en un solo boletín a  la vez, en cambio CVE si las identifica de forma única a cada una de ellas.
