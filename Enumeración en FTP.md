# ¿Qué es FTP?
FTP son las siglas de File Transfer Protocol es un estandar para la transferencia de ficheros y todos los servicios que están basados en este protocolo implementan una arquitectura de cliente-servidor y la implementan básicamente para que el cliente consulte o solicite recursos que se comparte en dicho servicio y este tipo de servicios son muy comunes.

Digamos que hago un escaneo con Nmap y me topo con un servicio que hace uso de FTP que normalmente corre en el puerto 21 (no tiene que ser así siempre) y para eso podemos usar la maquina "Anonymous" de TryHackMe

Para ver que es lo que se tiene que hacer cuando se hace un escaneo con Nmap y nos topamos con un servicio basado en un protocolo FTP como un ProFTPD o vsftpd.

Hay dos formas de enumerar:
1) Sin autentificación: Sin autentificación que es la enumeración mas básica podemos intentar conseguir la versión del servicio que está corriendo por ejemplo usando el parámetro "nmap -sV" y buscar por ejemplo si esa versión está desactualizada o tiene alguna vulnerabilidad conocida y si la tiene buscar exploits para poder obtener mas información o si es posible para directamente romperla
2) Con autentificación: para la forma autenticada primero tendríamos que acceder al servidor FTP y a partir de ahí ver que hay adentro e intentar conseguir mas información sobre su configuración o meramente sus recursos que están disponibles. Para poder acceder al servidor FTP hay formas como hacer un ataque de diccionario con una lista de posibles usuarios y una lista de posibles contraseñas, esto es una explotación de vulnerabilidades con credenciales débiles, para esto podríamos usar herramientas como Hydra o Medusa pero hay veces que podría no ser completamente necesario porque en los servidores FTP porque muchas veces lo suelen configurar mal porque para ese tipo de servicios existe un "Login Anónimo" que se puede activar o desactivar (por lo que muchas veces se deja activado) por lo que nos deja loguearnos como "Anonymous" y sin contraseña. Para comprobar esto podemos directamente intentar conectarnos vía FTP como usuario y sin contraseña ("ftp 10.10.231.64" y seguir el proceso de login) o podemos hacerlo mediante Nmap enviando un script FTP ("nmap --script ftp-anon -p21 10.10.231.64") lo que hará Nmap es intentar loguearse con este mismo usuario y sin contraseña para así en el resultado del escaneo decirnos si el login del usuario "Anonymous" está permitido o no. También podríamos hacer esto con "nmap -sC -p21 10.10.231.64" para enviar los scripts predeterminados de Nmap que en dicha categoria ya se incluye el script "ftp-anon". Si conseguimos acceso con el usuario anónimo lo primero que tenemos que ver es a que recursos tenemos acceso como dicho usuario porque puede que estemos limitados a ciertos recursos o directamente a ninguno pero dentro del login anónimo tenemos una serie de comandos que funcionan al nivel de los servicios que implementan el protocolo FTP, son comandos diferentes a los que tenemos en una Shell Bash para ver estos comandos podemos usar el comando "help" para mostrarnos los comandos disponibles  algunos son similares a los que tenemos en una Shell y otros son propios de FTP. Los posibles comandos que podríamos usar para temas de enumeración son "put" y "get", "get" nos permite traernos recursos del servidor victima a nuestra maquina atacante y "put" nos permite cargar recursos
